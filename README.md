# Anvil

Anvil is a multi-cloud Pulumi provider that wraps raw cloud resources (AWS, Azure, GCP) into opinionated, secure-by-default components. Instead of wiring up 5-10 individual resources for a single S3 bucket, you declare one `anvil:aws:Bucket` with a `dataClassification` and Anvil handles the rest вЂ” public access blocks, versioning, encryption, lifecycle rules вЂ” all baked in.

The provider is written in Go using the `pulumi-go-provider` SDK. SDKs are auto-generated for TypeScript, Python, Go, and .NET.

## Quick Start


## Build Pipeline

All commands run from the project root via `npm run`:

| Command                  | What it does                                                                                        |
| ------------------------ | --------------------------------------------------------------------------------------------------- |
| `npm run build`          | Full pipeline (all steps below in order)                                                            |
| `npm run generate`       | Fetches upstream schemas (e.g. AWS) and enriches component `schema.json` files with transform types |
| `npm run merge`          | Crawls all `provider/<cloud>/<resource>/schema.json` and merges into `provider/schema.json`         |
| `npm run registry`       | Crawls all `provider/<cloud>/<resource>/*.go` and generates `provider/cmd/anvil/main.go`            |
| `npm run build-provider` | Compiles the provider binary to `bin/pulumi-resource-anvil`                                         |
| `npm run gensdk`         | Generates SDKs from the merged schema                                                               |
| `npm run build:nodejs`   | Generate only the Node.js SDK                                                                       |
| `npm run build:python`   | Generate only the Python SDK                                                                        |
| `npm run build:go`       | Generate only the Go SDK                                                                            |
| `npm run build:dotnet`   | Generate only the .NET SDK                                                                          |

## Adding a New Resource

See [provider/README.md](provider/README.md) for the full guide.

The short version: create a folder under `provider/<cloud>/<resource>/` with a `schema.json` and a `.go` file, then run `npm run build`. Everything else is automatic.

## Local Development

To test locally, point your Pulumi app at the local binary:

```yaml
# Pulumi.yaml
plugins:
  providers:
    - name: anvil
      path: ../bin
```

## Contributing

Contributions are welcome. Please follow the fork-and-PR workflow:

1. **Fork** this repository to your own GitHub account
2. **Clone** your fork and create a feature branch off `master`
3. **Make your changes** вЂ” ensure `npm run build` completes successfully
4. **Push** to your fork and open a **Pull Request** against `master`
5. Wait for review вЂ” all PRs require maintainer approval before merge

Branch protection rules are enforced on `master`. Direct pushes, force pushes, and branch deletion are blocked. All PRs require code owner review and review thread resolution before merging.

Please keep PRs focused вЂ” one feature or fix per PR. If you're planning a larger change, open an issue first to discuss the approach.

# Anvil

Anvil is a multi-cloud Pulumi provider that wraps raw cloud resources (AWS, Azure, GCP) into opinionated, secure-by-default components. Instead of wiring up 5-10 individual resources for a single S3 bucket, you declare one `anvil:aws:Bucket` with a `dataClassification` and Anvil handles the rest — public access blocks, versioning, encryption, lifecycle rules — all baked in.

The provider is written in Go using the `pulumi-go-provider` SDK. SDKs are auto-generated for TypeScript, Python, Go, and .NET.

## Quick Start

```bash
# Full build: generate schemas → merge → register → compile → gen SDKs
npm run build
```

## Project Structure

```
anvil-core/
├── provider/                  # The Go provider (where go.mod lives)
│   ├── aws/                   # AWS components
│   │   ├── bucket/            #   S3 Bucket (with transforms)
│   │   │   ├── bucket.go      #     Component implementation
│   │   │   └── schema.json    #     Pulumi schema fragment
│   │   └── lambda/            #   Lambda Function
│   │       ├── lambda.go
│   │       └── schema.json
│   ├── azure/                 # Azure components
│   │   └── function/          #   Azure Function App
│   │       ├── function.go
│   │       └── schema.json
│   ├── cmd/anvil/             # Provider entrypoint (auto-generated)
│   │   └── main.go            #   ⚡ DO NOT EDIT — generated by registry script
│   ├── base-schema.json       # Skeleton schema (provider metadata)
│   ├── schema.json            # Merged master schema (auto-generated)
│   └── go.mod
├── scripts/                   # Build automation (all run from project root)
│   ├── generate/              #   Enriches component schemas from upstream providers
│   ├── merge/                 #   Merges all schema fragments into one master schema
│   └── registry/              #   Generates main.go with all component registrations
├── sdk/                       # Auto-generated SDKs
│   ├── nodejs/
│   ├── python/
│   ├── go/
│   └── dotnet/
├── bin/                       # Compiled provider binary
│   └── pulumi-resource-anvil
├── test-app-go/               # Go test app
└── package.json               # Build pipeline orchestration
```

## Build Pipeline

All commands run from the project root via `npm run`:

| Command                  | What it does                                                                                        |
| ------------------------ | --------------------------------------------------------------------------------------------------- |
| `npm run build`          | Full pipeline (all steps below in order)                                                            |
| `npm run generate`       | Fetches upstream schemas (e.g. AWS) and enriches component `schema.json` files with transform types |
| `npm run merge`          | Crawls all `provider/<cloud>/<resource>/schema.json` and merges into `provider/schema.json`         |
| `npm run registry`       | Crawls all `provider/<cloud>/<resource>/*.go` and generates `provider/cmd/anvil/main.go`            |
| `npm run build-provider` | Compiles the provider binary to `bin/pulumi-resource-anvil`                                         |
| `npm run gensdk`         | Generates SDKs from the merged schema                                                               |
| `npm run build:nodejs`   | Generate only the Node.js SDK                                                                       |
| `npm run build:python`   | Generate only the Python SDK                                                                        |
| `npm run build:go`       | Generate only the Go SDK                                                                            |
| `npm run build:dotnet`   | Generate only the .NET SDK                                                                          |

## Adding a New Resource

See [provider/README.md](provider/README.md) for the full guide.

The short version: create a folder under `provider/<cloud>/<resource>/` with a `schema.json` and a `.go` file, then run `npm run build`. Everything else is automatic.

## Local Development

To test locally, point your Pulumi app at the local binary:

```yaml
# Pulumi.yaml
plugins:
  providers:
    - name: anvil
      path: ../bin
```
